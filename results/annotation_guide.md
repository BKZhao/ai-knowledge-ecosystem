# 标注指南

## 分类定义

### Type 1: How-to（操作指引）
问题核心是"**如何完成某个具体操作**"，有明确的标准答案。
- 例：How do I sort a list in Python?
- 例：How to connect to MySQL database in Java?
- 关键词：how to, how do I, how can I

### Type 2: Debug（错误排查）
问题包含**具体的错误信息或异常行为**，需要找出原因并修复。
- 例：Getting TypeError: 'NoneType' object is not iterable
- 例：My loop runs infinitely, what am I doing wrong?
- 关键词：error, exception, not working, fails, why is this

### Type 3: Conceptual（概念理解）
问题是关于**概念的理解、比较或解释**，没有唯一标准答案。
- 例：What is the difference between process and thread?
- 例：When should I use REST vs GraphQL?
- 关键词：what is, difference between, why does, when should

### Type 4: Architecture/Design（架构设计）
问题涉及**系统设计、最佳实践或架构决策**，需要经验判断。
- 例：How to design a scalable microservice system?
- 例：What is the best database schema for a social network?
- 关键词：best practice, design pattern, architecture, should I use

## 标注说明
- 在 question_type 列填写数字（1/2/3/4）
- 在 type_label 列填写对应文字（可选）
- 如果不确定，选最接近的类型
- 每人负责200条，5人分工完成
