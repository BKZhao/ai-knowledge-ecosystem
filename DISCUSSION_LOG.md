## 2026-03-20 第一次完整讨论

### 今天确定的核心内容

**研究问题（定稿）：**
生成式AI工具的普及，是否对知识消费（SO提问）和知识生产（GitHub仓库创建）产生了方向相反的不对称冲击？

**核心发现：**
- SO消费端：-75.3%（ChatGPT后）
- GitHub生产端：+121.1%
- 不对称系数 β₂ = +3.82（p<0.001，双向固定效应）

**新发现（今天挖到）：**
- GitHub Issue/仓库比率大幅下降（TypeScript -54.4%，Python -37.3%）
- 意味着GitHub Issue和SO提问都在被AI替代（"协作式知识消费"也被替代）
- 统一理论：被动寻求帮助↓，主动创造↑

## 审稿人视角：当前论文的主要缺陷

### 🔴 致命缺陷（必须解决）

**缺陷1：GitHub数据质量问题**
审稿人会问：GitHub仓库+121%是真实知识生产增加，还是AI生成的垃圾项目涌入？
→ 解决方案：加入质量指标对比（活跃率/fork率/star率）

**缺陷2：ARI指数的内生性**
审稿人会问：你的AI可替代性指数（ARI）是你自己定的，不是客观数据。Python ARI=0.92的依据是什么？
→ 解决方案：引用HumanEval benchmark原始数据，或者用多种ARI测量做稳健性检验

**缺陷3：因果识别不够干净**
审稿人会问：β₂显著，但这可能只是因为GitHub本来就在增长（趋势），而不是因为ChatGPT。
→ 解决方案：加强平行趋势检验；用Copilot GA（2022.06）作为更早的干净事件做稳健性

### 🟡 重要问题（需要解决）

**问题4：样本量偏小**
2,390个面板观测（13语言×99月×2平台）对于发顶刊有点少
→ 解决方案：加入更多编程语言（GitHub有500+语言），或者降到周度面板

**问题5：机制不够清晰**
知道了"是什么"（不对称），但"为什么"不够深
→ 解决方案：加入机制检验（文本分析：问题类型变化；用户层面：新手vs老手分流）

**问题6：对照社区逻辑矛盾**
Math SE下降-60.9%，本来想用它作对照，但它自己也受到了冲击
→ 解决方案：换用"生活技能类"社区（Cooking SE等）；或者重新定位Math SE的角色（不是对照，而是"AI冲击更广泛"的证据）
