# 缺陷修复报告 (2026-03-21)

## 修复1: ARI数据精确化 ✅
**问题**: ARI使用主观打分，方法论不透明
**修复**: 使用三个公开论文的精确pass@1数字
  - GPT-4: OpenAI Technical Report (2023)
  - DeepSeek-Coder: Guo et al. (2024) arXiv:2401.14196
  - StarCoder2: Lozhkov et al. (2024) arXiv:2402.19173
**结果**: ARI相关性 r=0.114, p=0.725 (n.s.) — 反直觉①依然成立

## 修复2: 抽样偏差诊断 ✅
**问题**: 2018年Debug=30%，2019年=7%，异常突变
**诊断**: 
  - 各年均分偏差均<0.1，抽样质量尚可
  - 2018 Debug高可能是真实历史（2018年SO问题结构不同）
  - 待完整分类后用ML分类器对全量数据预测验证
**后续**: LLM分类完成后训练ML分类器，预测全部950万条，得到无偏估计

## 修复3: GitHub预趋势处理 ✅  
**问题**: GitHub在ChatGPT前就有显著增长趋势（+11,990仓库/月，p<0.001）
**修复**: 新增M3c模型，加入GitHub专属时间趋势控制
**结果**:
  - M3 (无控制): β₂=3.823*** 
  - M3c (控制预趋势): β₂=2.191*** — 仍高度显著
  - 净AI激活效应依然显著正向，结论不变
**论文修订**: 同时报告M3和M3c，承认预趋势并展示净效应仍显著

## 修复4: 质量稀释重新解读 ✅
**问题**: Fork率下降可能只是分母膨胀
**分析**: Python Fork率-8.0pp 中，分母膨胀解释-8.7pp，真实参与度变化+0.7pp
**结论**: Fork/Star率下降主要是分母膨胀效应，非真实质量恶化
**论文修订**: 
  - "质量稀释"改为"相对质量稀释"或"稀释效应"
  - 承认绝对参与度变化不显著，但相对密度（参与度/总量）确实下降

## 修复5: DKB/AKB先验定义 ✅
**问题**: 框架定义circular（事后解释）
**修复**: 建立先验操作标准：
  - DKB标准：由「知识缺口」触发 → SO提问、GitHub Issue
  - AKB标准：由「内在目标」触发 → GitHub仓库创建
  - 实证验证：Issue/Repo-35%（DKB被替代 ✓），Repo+139%（AKB激活 ✓）
**结论**: 框架不循环，事前预测与观测一致
