# SO 2024-2025 数据补充操作指南

## 项目地址

**GitHub**: https://github.com/BKZhao/ai-knowledge-ecosystem

需要运行的脚本：
```
stackoverflow_research/fetch_so_2024plus.py
```

---

## 背景

项目研究生成式AI对知识生态系统的冲击。目前大部分数据已截止到2025年12月，但有两项数据只到2024年3月：

| 数据项 | 当前范围 | 需补充 |
|--------|---------|--------|
| SO质量指标 | 2018-01 ~ 2024-03 | 2024-04 ~ 2025-12 |
| LLM分类样本 | 2018-01 ~ 2024-03 | 2024-04 ~ 2025-12 |

原因：SO官方数据转储只到2024年3月，后续数据需要通过SE API抓取。

---

## 操作步骤

### 第一步：克隆项目

```bash
git clone https://github.com/BKZhao/ai-knowledge-ecosystem.git
cd ai-knowledge-ecosystem/stackoverflow_research
```

### 第二步：安装依赖

```bash
pip install pandas requests pyarrow
```

### 第三步：运行抓取脚本

```bash
python fetch_so_2024plus.py
```

脚本会提示按回车确认，然后开始抓取。

**预计耗时**：2-3小时（21个月 × 14语言 × 每月若干次API调用）

### 第四步：确认输出文件

抓取完成后，检查 `data/fetched_2024plus/` 目录，应该有以下文件：

```
data/fetched_2024plus/
├── monthly_stats.csv      # 月度质量统计（约300行）
├── questions_sample.csv   # 全部问题（约10-15万条）
└── llm_sample.csv         # LLM分类抽样（约1.5万条）
```

### 第五步：上传结果

将 `data/fetched_2024plus/` 目录打包：

```bash
cd data
tar -czvf fetched_2024plus.tar.gz fetched_2024plus/
```

然后发给炳坤，或上传到服务器：

```bash
scp fetched_2024plus.tar.gz user@服务器IP:/path/to/stackoverflow_research/data/
```

## 脚本说明

### 抓取内容

- **时间范围**：2024年4月 ~ 2025年12月（21个月）
- **编程语言**：14种（Python, JavaScript, TypeScript, Java, C#, PHP, C++, Swift, Kotlin, Go, Rust, Scala, Haskell, R）
- **API**：Stack Exchange API（已内置API Key）

### API限制

- 有Key：10,000次请求/天
- 预计需要：~300次请求（21月 × 14语言）
- 脚本已内置延时，不会触发限制

### 输出说明

1. **monthly_stats.csv** - 每月每语言的质量指标：
   - `n_questions`: 问题数
   - `avg_score`: 平均得分
   - `avg_views`: 平均浏览量
   - `pct_answered`: 回答率

2. **llm_sample.csv** - 用于LLM分类的问题样本：
   - 包含 `title` 和 `body`（问题正文）
   - 分层抽样：每月每语言最多50条
   - 用于四分类（How-to, Debug, Conceptual, Architecture）

## 常见问题

### Q: 如果中途断了怎么办？

脚本每次抓取一个语言，如果中断可以手动修改脚本从断点继续。或者直接重新运行（会覆盖已有文件）。

### Q: IP被封怎么办？

脚本设计了0.1秒延时，正常情况不会触发。如果被封，可以：
1. 等待24小时后重试
2. 使用代理
3. 换个网络环境

### Q: 需要多少存储空间？

预计输出文件总共约50-100MB。
